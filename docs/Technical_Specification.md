# Технічне завдання на розробку веб-системи «AutoRent»
1. Найменування та область застосування
Найменування:
Веб-система «AutoRent» — онлайн-платформа для оренди автомобілів.
Область застосування:
Використання в Україні для фізичних осіб, туристів та бізнес-клієнтів, що потребують короткострокової чи довгострокової оренди авто.
2. Підстава для розробки
Розробка проводиться в межах дисципліни «Основи програмної інженерії» студентом спеціальності ІПЗ.
Документ створено відповідно до:
- ISO/IEC 29148:2018
- ISO/IEC 25010:2011
- Agile Manifesto
- Scrum Guide 2020
3. Призначення розробки
Функціональне призначення:
Система забезпечує:
- пошук та бронювання автомобілів онлайн;
- управління профілями клієнтів;
- адміністрування автопарку;
- формування договорів та фінансових звітів.
Експлуатаційне призначення:
Веб-додаток з доступом через браузери (Chrome, Firefox, Safari). Адаптований для використання на ПК, планшетах та смартфонах.
4. Технічні вимоги
4.1. Вимоги до складу програмного продукту
- Модуль автентифікації (JWT + OAuth 2.0)
- Модуль пошуку та фільтрації авто
- Модуль бронювання та оплати
- Модуль керування профілями
- Модуль адміністратора (автопарк, замовлення)
- API для інтеграції з платіжними системами
4.2. Вимоги до функціональних характеристик
 №    Функція                 Опис                                                         
-----------------------------------------------------------------------------------------
 1   Реєстрація              Створення облікового запису клієнта                         
 2   Пошук авто              Фільтрація за датами, типом авто, ціною                     
 3   Бронювання              Оформлення замовлення з вибором додаткових послуг           
 4   Онлайн-оплата           Інтеграція з платіжними системами (LiqPay, Stripe)          
 5   Управління замовленнями Перегляд історії, скасування, продовження оренди            
 6   Звітність               Формування звітів для адміністратора (PDF/Excel)            

4.3. Вимоги до надійності
- Автозбереження даних кожні 30 секунд
- Резервне копіювання бази даних щоденно
- Обробка помилок платіжних транзакцій
- Захист від DDoS-атак

7. Порядок контролю та приймання
Види тестування:
- Unit-тести (Jest)
- Integration-тести (Postman) 
- UI-тести (Playwright)
- Користувацьке тестування через Google Forms
Приймання ПЗ вважається успішним, якщо:
- проходить 100% тестів;
- дані зберігаються коректно;
- всі основні функції доступні через веб-інтерфейс.

9. Очікуваний результат
- Працюючий прототип веб-сайту оренди авто
- Тестові акаунти для перевірки функціоналу
- Повна документація по проекту
Ціль: Спростити та автоматизувати процес оренди автомобілів.
Користувачі: Туристи, бізнесмени, місцеві жителі (18-60 років).
Проблема: Складність швидкого пошуку та бронювання авто, недостатня прозорість цін, великі витрати часу.

**Рішення:** Зручний онлайн-сервіс з миттєвим підбором авто, прозорими тарифами та швидким бронюванням.
